# 內科專科題庫 — 題目灌入規格

本 repo 是單檔 React 題庫（`index.html` + `questions.json`），部署於 GitHub Pages。
你的任務：把 `resource/` 內各年度 PDF 的題目與答案，解析成符合下列格式的題目物件，寫入 `questions.json`。

**本題庫全部為單選題（`type:"S"`），沒有複選題。所有題目的 `answer` 一律為單一字串。**

**本階段只做「題目 + 答案」，不寫詳解（explanation）。詳解之後另行處理。**

---

## 資料來源

- PDF 位於 `resource/`，檔名形如 `109 年 內科專科.pdf` … `114 年 內科專科.pdf`。
- **答案印在每題題號旁邊**（例如「1.(C)」或題號後標注），直接採用 PDF 標示的答案，不要自行重新作答或「修正」答案。

---

## questions.json 格式

`questions.json` 是一個 JSON 陣列，每個元素是一題：

```json
{
  "id": "109-S1",
  "year": 109,
  "type": "S",
  "question": "題幹完整文字……",
  "choices": [
    { "key": "A", "text": "選項 A 文字" },
    { "key": "B", "text": "選項 B 文字" },
    { "key": "C", "text": "選項 C 文字" },
    { "key": "D", "text": "選項 D 文字" }
  ],
  "answer": "C"
}
```

### 欄位規則

- `id`：格式 `{年}-S{流水號}`，例：`109-S1`、`112-S3`。流水號依 PDF 題序從 1 起算。
- `year`：數字（109–114）。
- `type`：一律為 `"S"`（單選）。本題庫不含複選題。
- `question`：題幹完整文字。保留原始換行語意但去除 PDF 斷行造成的多餘換行（同一句被 PDF 換行切斷者要接回）。
- `choices`：陣列，每項 `{ "key": "A"|"B"|"C"|"D"|"E"…, "text": "…" }`。key 用大寫字母，依 PDF 順序。
- `answer`：**單一字串**，例 `"C"`。（全為單選題，不會是陣列。）
- **不要**加 `explanation` 欄位（本階段略過）。

---

## 嚴格規則

1. **反向題保留原樣**：題幹含「為非」「不正確」「何者錯誤」「不妥」「並非」「下列何者不」等否定語的題目，**逐字保留**，不可改寫成正向題，答案沿用 PDF 標示。
2. **答案忠實照抄**：一律採用 PDF 題號旁的答案。即使你認為答案有誤，也不要更動；若懷疑有誤，列入低信心清單（見下）。
3. **不臆造內容**：題幹、選項、答案任何一處在 PDF 中模糊、缺漏、無法判讀時，**不要猜**，列入低信心清單並在該題暫時保留你最佳判讀，但明確標記。
4. **不重新編號既有題目**：若 `questions.json` 已有題目，新增時延續流水號，不要打亂既有 id。
5. **全部視為單選**：若某題在 PDF 中疑似為複選（多個答案標示），不要自行建成 `type:"M"`，改列入低信心清單由人工裁定。

---

## 重複偵測

寫入前先比對是否與現有題目重複（同年度跨批次灌入時）。重複判斷：
- 第一層：題幹去除空白後完全相同 → 視為重複，跳過。
- 第二層：同 `year` + 題幹前 30 字相同 → 標記為「疑似重複」列入清單，由人工裁定，不自動寫入。

---

## 低信心清單（必須產出）

每次解析後，在 repo 根目錄產生／更新 `review_needed.md`，列出需人工校對的題目，至少涵蓋：
- 答案在 PDF 中抓不到或格式異常的題
- 題幹疑似被截斷（結尾無標點、語意不完整）
- 選項數異常（少於 3 或多於 6）
- 疑似為複選（PDF 標示多個答案）的題
- 含圖片／表格、純文字可能不完整的題
- 疑似重複題

格式：每題一列，含 `id`、年度、問題類別、原因摘要。

---

## 流程

1. 解析指定年度 PDF（一次處理一個年度，除非我指定多個）。
2. 產生該年度題目物件，做重複偵測。
3. 合併進 `questions.json`（保持陣列、UTF-8、繁體中文不轉義）。
4. 跑驗證（見下），確認通過。
5. 更新 `review_needed.md`。
6. **先停下來回報**：本次新增幾題、低信心幾題。**等我確認後**再執行 commit / push。

---

## 驗證

合併後務必跑一次完整性檢查（可用 node 或 python），確認：
- `questions.json` 是合法 JSON。
- 每題具備 `id`、`year`、`type`、`question`、`choices`、`answer`。
- 每題 `type` 皆為 `"S"`，`answer` 皆為字串（不得出現陣列）。
- `answer` 的 key 存在於該題 `choices` 的 key 集合內。
- `id` 全域唯一。
- `choices` 每項都有非空 `text`。

任一項失敗 → 不要 commit，回報問題。

---

## Git 慣例

- 確認後才 commit。commit message 用英文，描述本次內容，例：`Add 109 questions (80 items)`。
- push 指令為單一不含註解的程式碼區塊。
- **絕不**把 `resource/` 或 PDF 加入 git（已在 `.gitignore`）。

---

## 更新快取版本號

每次成功灌入並 push 後，將 `index.html` 中載入字串 `questions.json?v=N` 的版本號遞增（`v=1`→`v=2`…），避免使用者抓到舊快取。此修改與題庫一起 commit。
